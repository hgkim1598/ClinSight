# 중환자실(ICU) 임상 피드백 정리

> **출처**: 중환자실 근무 의료진 인터뷰 기반  
> **목적**: 패혈증 환자 모니터링 대시보드 UI/UX 개선을 위한 피드백 구조화  
> **범례**: 🔍 = 필기 해독이 어려운 부분(`?` 표시)을 근거 기반으로 보완한 해설

---

## 1. 환자 목록 화면 (Patient List)

### 1-1. 과(科)별 분리와 담당의 표시

- 중환자실 내에는 **여러 진료과**가 공존하지만, **각 과는 자기 환자만 본다**.
- 따라서 환자 목록에 **소속 과(科)와 담당 의사**를 반드시 표시해야 한다.
- **병상점유율** 같은 전체 지표보다는 **과별 환자 수**가 더 유용하다.
- 로그인한 의료진 기준으로 **자기 환자 리스트를 상단에 정렬**해주는 것이 좋다.
- 담당 의사가 누구인지 확인할 수 있어야 → 필요시 **담당 의사 호출(콜)**이 가능하다.

### 1-2. 입실 일수 표시

- 환자 리스트에서 **중환자실 입실 시점 기준 재원 일수**(며칠째인지)를 함께 표시하면 좋다.
- 병원 내부 앱에서도 다음 정보들을 확인한다:
  - 담당의, 재원 병동, 입원일자
  - **HOD (Hospital Day)**: 입원 일수. 입원한 날을 1일로 카운트  
  - **POD (Postoperative Day)**: 수술 후 일수. 수술 당일은 POD #0, 다음 날이 POD #1  
  - V/S (활력징후), 검사 결과, 기록 조회, 진료이력, 처방 조회

> 🔍 **"레저턴트가 정해져있고 턴이 정해져있다(중환자실 - 응급실 - 입원병동)"**  
> → **레지던트(Resident) 순환근무(Rotation)** 이야기로 추정된다. 수련의(레지던트)는 일정 주기로 중환자실 → 응급실 → 일반병동을 돌아가며 근무한다. 예를 들어 4주 단위 블록으로 ICU 2블록, 일반병동 5블록, 응급실 1블록 같은 스케줄을 소화한다(미국 내과 수련 프로그램 사례 기준). 한국 병원도 유사하게 레지던트가 파트를 돌아가며 배치되며, 당직(턴)이 정해져 있다.

### 1-3. 화면 방향

- 태블릿(패드) 기준 **세로가 자연스럽겠지만**, 현재 UI는 **가로가 더 낫다**는 평가.
- 가로로 봐야 하면 "알아서 돌리면 된다"는 의견.

## 2. 활력징후 (Vital Signs, V/S)

### 2-1. 기본 활력징후 세트

다음 5가지는 **하나의 세트**로 함께 봐야 한다:

| 항목 | 영문 | 단위 | 참고 |
|------|------|------|------|
| 혈압 (Blood Pressure) | BP (SBP/DBP) | mmHg | 수축기/이완기 |
| 심박수 (Heart Rate) | HR | 회/분 (bpm) | **정수값**으로 표시. 소수점 없음 |
| 호흡수 (Respiratory Rate) | RR | 회/분 | 아래 별도 설명 |
| 산소포화도 | SpO₂ | % | **95~100%** 범위, 소수점 없음, 100이 최대 |
| 체온 (Temperature) | Temp | °C | 감염 시 발열 모니터링 핵심 |

### 2-2. 혈압: 일반 혈압 + 동맥압(ABP) 병행

- 중환자실에서는 **일반 혈압(커프식, NIBP)**과 **동맥 내 직접 측정(침습적 동맥압, IBP/ABP)**을 같이 본다.
- **동맥라인(A-line)**을 삽입하면 실시간 연속 혈압 측정이 가능하지만, 라인이 없으면 측정 불가.
- 동맥라인만 잡히거나 커프로 잘 측정이 안 되는 경우가 있으므로 **둘 다 표시**가 필요.

> 🔍 **"MAP는 라인이 없으면 잴 수가 없다"**  
> → **MAP (Mean Arterial Pressure, 평균동맥압)**은 계산 공식(MAP = (수축기 + 이완기×2) ÷ 3)으로 추정할 수 있지만, **가장 정확한 측정은 동맥라인을 통한 침습적 모니터링**이다. 패혈성 쇼크에서 MAP ≥ 65mmHg 유지가 치료 목표이며(Surviving Sepsis Campaign 가이드라인), 승압제(vasopressor) 투여 중 MAP을 90% 이상 동맥라인으로 모니터링한다는 대규모 ICU 연구 결과가 있다. 맥락상 "동맥라인 없이는 정확한 MAP 연속 모니터링이 어렵다"는 의미로 해석된다.

### 2-3. 호흡수 관련

- 호흡수는 따로 분리해서 보지는 않는다.
- 1분 동안 직접 세야 하지만, 제대로 체크 안 되면 **18로 통일**해서 기록하는 경우가 있다(현장 관행).
- **Trend 탭에서 V/S 세트의 일부로 함께 보면** 충분하다.

### 2-4. 측정 주기

- 활력징후는 기본적으로 **1시간마다** 입력됨.
- 혈압은 경우에 따라 **30분마다** 재기도 함.
- Trend도 **1시간 단위**로 보고, **24시간(전날 값 포함)** 보여주는 것 괜찮다는 의견.
- 측정 시점에 떠 있는 값(산소포화도, 혈압 등)은 **한 번에 같이 기록**된다.

### 2-5. 의식 상태 (GCS)

> 🔍 **GCS (Glasgow Coma Scale, 글래스고 혼수 척도)**는 1974년 영국 글래스고 대학에서 개발된 의식 수준 평가 도구다. **눈뜨기 반응(E, 1~4점) + 언어 반응(V, 1~5점) + 운동 반응(M, 1~6점)**의 합산으로 3~15점 범위이며, 15점이 완전 의식, 7점 이하는 혼수(coma)로 판단한다. ICU와 응급실에서 표준적으로 사용된다(StatPearls, NCBI).

## 3. 검사 수치 (Lab Values)

### 3-1. 수치 표시 규칙

- 대부분의 수치는 **소수점 없이 정수로** 표시. 특히 Heart Rate.
- 현재 mock 데이터에 소수점이 많이 들어가 있으므로 수정 필요.
- 검사 결과가 안 나오면 **X** 표시.

### 3-2. 과(科)별로 주로 보는 검사 항목

> 🔍 표시 부분은 필기 해독이 어려웠던 부분을 의학적 맥락으로 보완한 것임

| 분류 | 주요 검사 항목 | 설명 |
|------|---------------|------|
| **간기능 (Hepatic, HEP)** | OT/PT (AST/ALT) | 간 효소 수치. 간 손상 정도 평가 |
| **응고 (Coagulation, COAG)** | aPTT, INR | 혈액 응고 기능. 출혈/혈전 위험 평가 |
| **신기능 (Renal)** | eGFR, BUN, Cystatin-C | 신장 여과 기능 평가 (아래 상세 설명) |
| **호흡 (Respiratory, RESP)** | 현재 UI 유지 OK | - |
| **심장 (Cardio)** | pro-BNP, CK-MB, Troponin-T | 심부전·심근 손상 마커 |

> 🔍 **"hep에서는 otpt 많이 봄 (신장)"**  
> → OT/PT는 AST(GOT)/ALT(GPT)를 의미하며, 이건 간기능 지표다. 괄호 안의 "신장"은 아마 다음 줄의 Renal 파트와 혼재된 필기로 보임.

> 🔍 **"EDFR" (Renal 항목)**  
> → **eGFR (estimated Glomerular Filtration Rate, 추정 사구체여과율)**의 오기로 추정. 신장이 혈액을 얼마나 잘 걸러내는지를 나타내며, 단위는 mL/min/1.73m². 이 수치가 낮으면 신기능 저하를 의미한다.

> 🔍 **"신장 문제인지 아닌지 이게 방광쪽 문제인지…"**  
> → BUN/Creatinine, eGFR, Cystatin-C를 종합해서 **상부 요로(신장) 문제인지 하부 요로(방광/요도) 문제인지** 감별한다는 맥락. Cystatin-C는 크레아티닌보다 근육량 영향을 덜 받아 **신기능 변화를 더 빨리 감지**하는 바이오마커다(PMC 연구: 패혈증 환자에서 Cystatin-C가 크레아티닌보다 3시간 먼저 신손상을 반영).

> 🔍 **"se? pro-bnp" (Cardio)**  
> → **serum pro-BNP** 또는 **NT-proBNP**로 추정. 심부전 중증도 평가에 쓰이는 혈액 마커.

### 3-3. 자주 보는 전체 수치 목록

피드백에서 언급된 주요 검사 항목들을 정리하면:

| 항목 | 의미 |
|------|------|
| Hemoglobin (Hb) | 혈색소. 빈혈 여부 |
| Platelet (PLT) | 혈소판. 출혈/응고 능력 |
| Hematocrit (Hct) | 적혈구 용적률 |
| BUN / Creatinine (Cr) | 신기능 기본 지표 |
| eGFR | 추정 사구체여과율 (신기능 종합) |
| FBS | 공복혈당 (Fasting Blood Sugar) |
| OT/PT (AST/ALT) | 간 효소 |
| Uric acid | 요산 (통풍, 신기능 관련) |
| Na / K | 나트륨 / 칼륨 (전해질) |
| CRP | C-반응성 단백 (염증 수준) |
| Cystatin-C | 신기능 정밀 바이오마커 |
| ABGA | 동맥혈 가스 분석 (산-염기, 산소/이산화탄소) |
| Lactic acid | 젖산 (조직 관류 부족, 쇼크 지표) |
| CK-MB | 심근 효소 (심근경색 마커) |
| Troponin-T | 심근 손상 특이 마커 |
| pro-BNP | 심부전 마커 |
| Procalcitonin | 세균 감염 및 패혈증 특이 마커 |
| aPTT, INR | 응고 기능 |
| BST (혈당) | Blood Sugar Test. 아래 상세 |

### 3-4. 혈당 모니터링 (BST)

> 🔍 **"BST → 6A/10A/4P/9P30"**  
> → BST(Blood Sugar Test)를 하루 **4회** 정해진 시간에 측정한다는 의미:  
> - **6:00 AM** (6A)  
> - **10:00 AM** (10A)  
> - **4:00 PM** (4P)  
> - **9:30 PM** (9P30)  
>
> 이는 한국 병원에서 흔히 사용하는 혈당 측정 스케줄이다. 결과에 따라 인슐린 용량 코멘트를 주어 조절하기도 한다.

### 3-5. 검사 주기

- **피검사(혈액검사)**: 기본 하루 한 번. 필요에 따라 12시간 또는 6시간마다.
- **X-ray(흉부)**: 매일 촬영.
- 검사 시점에 떠 있는 활력징후 값을 함께 기록한다.

## 4. I/O (Intake/Output, 섭취량/배출량)

### 4-1. 개념

- **Intake**: 환자에게 들어간 모든 것 = 수액 + 약물 + 주사 + 식사 + 물
- **Output**: 환자에서 나온 모든 것 = 소변 + 배변 등
- **정상적으로는 1:1 비율**이 이상적.
- 이뇨제를 쓰면 output이 더 나오는 등, **치료에 따라 달라짐**.
- **마이너스(-)값도 정상일 수 있다** (의도적 I/O 조절 시).
- 일정 수준 이상 차이가 나면 문제로 봐야 한다.

### 4-2. 패혈성 쇼크와 I/O

- 패혈성 쇼크(septic shock)에서는 **소변량이 줄어든다** → 쇼크 상태 확인 근거 중 하나.
- 중환자실은 금식이 기본이라 **수액을 많이 넣는다** (영양과 단백질 공급 목적).
- 수액을 많이 넣다 보면 **폐에 물이 차서(폐부종) 붓는 경우**가 많다.
- 그래서 I/O 모니터링이 매우 중요하다.

### 4-3. UI 반영 사항

- **I/O 탭을 별도로 추가**해달라는 요청.
- Trend에서 **V/S 탭 + I/O 탭 + 활력징후 탭 + 혈당(BST) 탭**이 필요.

## 5. 배양 검사와 항생제 (Culture & Antibiotics) — 핵심 피드백

### 5-1. 배양 검사의 중요성

**의료진들은 배양 검사 결과를 가장 기다린다.** 이유:

1. 소변/혈액/가래 등에서 **균 배양 검사**를 시행.
2. 예: 요로 감염으로 소변에서 대장균이 나왔는데, **혈액에서도 같은 균**이 나오면 → **패혈증** 확정 (소변 → 혈액으로 균이 타고 들어간 것).
3. 배양 결과가 나오면 **항생제 감수성(Antimicrobial Susceptibility Testing, AST)** 결과를 함께 확인.
4. 감수성 검사에서 **MIC(최소억제농도) 수치가 더 작은 항생제**가 효과가 좋은 것 → 그 약을 선택.

> 🔍 배양 검사에서 균 동정(identification)까지 보통 24~48시간, 감수성 검사까지 48~96시간 소요(Frontiers in Medicine, 2021). 이 시간 동안은 경험적 광범위 항생제를 투여하며, 결과가 나오면 적절한 항생제로 전환(de-escalation)한다.

### 5-2. 배양 검사 추이가 중요

- **균이 나왔다 → 치료했다 → 이제 안 나온다** = 호전.
- **언제** 자랐고 **언제** 안 자랐는지 = 시간축 추이가 핵심.
- 중간에 안 나오는 것은 호전된 것.
- 다만 안 자란다고 항생제를 바로 중단하지는 않음.

### 5-3. 항생제 사용 이력 UI

- **항생제를 며칠부터 며칠까지 어떤 약을 썼는지** 타임라인으로 표시.
- 행(row) 헤더에 날짜, 열(column) 헤더에 약 이름, 언제 투약했는지 보여주기.
- 하단에 **항생제 감수성 결과** 표시 ("이 약이 이 균에 효과가 있다/없다").

### 5-4. 배양 검사 UI 버튼

- 버튼을 누르면 **배양검사 결과 화면**으로 이동하는 기능 요청.

> 🔍 **"감염내과 의존도가 높다? 항생제 처방 관련?"**  
> → 중환자실에서 패혈증 환자의 항생제 선택/변경은 **감염내과 전문의 자문(협진)**에 크게 의존한다는 의미. 배양 결과 해석, 내성 패턴 분석, 약제 변경 등에서 감염내과 역할이 크다.

> 🔍 **"종규기간???"**  
> → **"투여 기간" 또는 "항생제 투여 종료 기한"**으로 추정. 항생제를 며칠 투여할지, 언제 종료하거나 변경할지에 대한 정보로 보임.

> 🔍 **"배양검사 재시행?"**  
> → 치료 중간에 배양검사를 다시 하여 **균이 여전히 자라는지 확인**하는 과정. 균이 사라졌는지(호전) 아직 남아있는지(치료 실패 또는 지속 감염) 판단 근거.

## 6. Trend 기능

### 6-1. 기존 기능 유지 및 추가 요청

- 이전처럼 **여러 항목을 한 번에 비교하는 기능** 필요.
- Trend는 체온보다 **활력징후 전체 세트**를 보는 것이 더 중요.

### 6-2. 필요한 Trend 탭 구성

1. **V/S (활력징후) 탭**: 혈압 + 심박 + 호흡 + 산소포화도 + 체온
2. **I/O 탭**: 섭취량 / 배출량
3. **혈당(BST) 탭**

### 6-3. Onset 일수

- **Onset 일수**(발병 또는 증상 시작일로부터 며칠째)를 표시해주면 좋다.

---

## 7. 협진 요청 (Consultation Request)

### 7-1. 단일 수신자 선택

- 협진 요청은 같은 내용으로 **여러 명에게 보내지 않는다**.
- **한 명만 선택 가능**하게 UI를 변경해야 함 (현재 UI가 여러 과의 여러 명 선택 가능한 구조라면 수정 필요).
- 이미 선택된 상태에서 다른 사람을 선택하면 **변경(교체)**되어야 한다.
- 이유: 여러 명에게 협진이 가면 → 외래도 그 사람들한테 받아야 하고 → 의견이 여러 개 생겨서 → 의사들이 싫어한다.

### 7-2. 의뢰 내용 양식

```
상기환자는 (나이)세 (성별)환자, (진단명)으로 입원함
(상세 내용 기술)
고신 선처 부탁드립니다
감사합니다
```

> 🔍 **"고신 선처"**  
> → 한국 의료계에서 협진 의뢰서에 쓰는 관용 표현. "살펴봐 주시고 잘 처리해 주시기 바랍니다" 정도의 의미.

### 7-3. 긴급도 구분

- **스케줄 응급**: "내가 급하다" (의료진 기준의 긴급)
- **환자 응급**: "당장 환자가 죽을 것 같다" (생명 위급)

> 🔍 **"정수가 릴가?"**  
> → 해독 불가. 문맥상 긴급도 관련 기준이나 점수 체계에 대한 언급일 수 있으나 확실하지 않음.

## 8. 알람 시스템

### 8-1. 확인 버튼 필수

- 알람에는 반드시 **"확인" 버튼**이 있어야 한다.
- **책임 소재를 명확히** 하기 위함: "나는 분명 알렸는데 네가 보지 않았다"를 방지.
- 현재 병원에서도 검사 결과 → 교수님 문자 → **확인 표시로 숫자를 입력**하게 하는 방식 사용.

## 9. 사망 위험도 표시 (High/Med/Low)

- 병원에서 실제로 사망 **'위험률'을 퍼센트(%)로 매기지는 않는다**.
- 현재 UI 기준이 괜찮다는 의견 (개인적으로 50% 이상이 High라고 생각하지만, 그렇게 하면 대부분 빨간색이 됨).
- **현재 기준을 유지하고, 의료진이 수치를 직접 보고 판단**하도록 하는 게 낫다.

## 10. 메모/인계 기능 (Notes)

### 10-1. 당직 인계 메모

- 24시간 근무가 아니라 **당직 교대** 시스템이므로, 다음 당직에게 **인계할 수 있는 메모장** 역할 필요.
- 기록할 내용 예시:
  - "무슨 검사 주의해서 봐라"
  - "어떤 약을 쓴 이유"
  - 주요 관찰 사항

### 10-2. 좌측 노트 활성화

- 시간이 되면 **좌측 노트 영역도 활성화**해달라는 요청.

## 11. 챗봇 기능

### 11-1. 의사들의 AI 활용 현황

- 의사들도 **GPT를 많이 쓴다**.
  - "진단서 써줘"
  - 논문이나 연구 계획서 작성
  - 신손상 시 항생제 용량 문의

### 11-2. 항생제 용량 계산 (핵심 유스케이스)

- **UpToDate**(유료 임상 의사결정 지원 사이트)에서 원래 계산 관련 정보를 가져옴.
- 핵심 질문: "이 환자는 몸무게가 몇 kg이고, 신기능 수치가 얼마인데, 항생제 어떻게 줄까?"
- 필요 정보: **키, 몸무게, 신기능(eGFR/GFR 기준)** → kg당 용량 계산 → n시간마다 투여
- 약이 정해지면 구체적으로: 예) "4.5g씩 6시간마다"
- 환자 페이지 내 챗봇이라면 **환자 데이터를 이미 프롬프트에 넣을 수 있으므로** 다시 찾아볼 필요 없이 바로 답변 가능.

> 🔍 **항생제 신기능 기반 용량 조절**은 실제 임상에서 매우 중요한 부분이다. GFR에 따라 항생제 용량을 줄이거나 투여 간격을 늘리는 것이 표준 관행이며, 이를 잘못하면 약물 축적으로 부작용이 생기거나 반대로 치료 실패가 발생한다(UCSF Antimicrobial Dosing Guidelines 등 참고).

### 11-3. 보험 관련 정보

- 피검사 수치 여러 개 중 **3개에 해당하면 보험/비보험이 갈리기도** 함.
- 이런 정보도 챗봇이 제공하면 유용할 것.

### 11-4. 약 관련 참고

- 병원에 **있는 약, 없는 약, 잘 안 쓰는 약** 정보도 있으면 좋다.

> 🔍 **"APTT 이런거 보기 좋아?"**  
> → aPTT(활성화 부분 트롬보플라스틴 시간)는 응고 기능 검사. 맥락상 "챗봇에서 aPTT 같은 수치도 바로 볼 수 있으면 좋겠다"는 의미로 추정.

## 12. 환자용 대시보드에 대한 의견

**"환자용 대시보드는 절대 필요 없다."** 이유:

1. 나아지고 있는 환자인데 현재 확률/수치가 높아 보이면 **오히려 오해와 불안**을 유발.
2. 중환자실 패혈증 합병증 환자는 **의식이 없는 경우가 많다**.
3. 고령 환자가 많아 대시보드를 본다 해도 이해가 어려움.
4. 단, 현재 **상세 화면들은 보호자 설명 시 유용**할 수 있음 → 정확한 수치로 보여주면 **납득하기 쉬움**.

## 13. 의무기록 및 경과 기록 관련

### 13-1. 의사가 매일 확인하는 것

- "오늘 어때?" → 혈압 / 피검사 등 경과 확인.
- 과거력(고혈압 등 만성질환) 유지.
- 자신이 어떤 조치를 했는지 기록: "혈뇨를 봤다 → 내가 뭘 했으니 → 이상이 있으면 뭘 하자"

### 13-2. 한 번에 볼 수 있으면 좋겠다는 요청

- 현재는 "~때문에 ~하고 있고 ~때문에 ~부탁한다"의 **근거를 보려면 UI를 타고타고 들어가야** 함.
- **한 화면에서 핵심 정보를 모아 볼 수 있으면** 좋겠다.

### 13-3. 앱 사용 패턴

- 병원 내부 앱도 **회진 가기 전에 보고 가는 용도**. 회진 중에 앱을 켜지는 않음.

### 14-1. 용어 수정

## 15. CVA 관련

> 🔍 **"CVA - 뭐때문인지, 띄운 시간"**  
> → **CVA (Cerebrovascular Accident) = 뇌졸중(뇌혈관 사고)**. 뇌졸중이 발생했을 때 **원인(허혈성/출혈성)**과 **증상 발생 시각(onset time)**을 기록하는 것이 매우 중요하다. 특히 허혈성 뇌졸중은 발생 후 4.5시간 이내에 혈전용해제(tPA) 투여가 가능하므로, "띄운 시간"은 증상 onset 시각을 의미하는 것으로 보인다(NCBI StatPearls).

## 16. 기타 참고 사항

### 16-1. 몸무게 측정

- 중환자실은 **침대 자체에 체중 측정 기능**이 있음.

### 16-2. 순천향대 병원 참고

- **딥카스(DeepCars)** 업체 서비스를 사용하여 심전도 등을 큰 모니터에 vital과 함께 띄워둠.

### 16-3. 영상 검사

- 중환자실에서는 **CT와 흉부 X-ray를 무조건 찍는다**.

> 🔍 **"요로감염 띄우기?"**  
> → 요로 감염(UTI)이 발생했을 때 이를 UI에 알림이나 표시로 띄워주는 기능에 대한 언급으로 추정.

> 🔍 **"호흡기 쪽은 res renal ??"**  
> → 문맥이 불분명. "RESP(호흡기)와 RENAL(신장) 파트를 함께 보는 상황"이거나, 호흡기계와 신장계가 연관된 검사 항목에 대한 언급일 수 있음.

> 🔍 **"전체 알문? 뜨는데 그렇게까지 할 필요는 없다"**  
> → "전체 알림이 뜨는데 그렇게까지 할 필요는 없다"로 추정. 항생제 사용 이력에서 전체 알림을 뜨게 하는 것은 과하다는 의견.

> 🔍 **"약물 복용력"**  
> → 환자의 기존 약물 복용 이력도 대시보드에서 볼 수 있으면 좋겠다는 의견으로 추정.
